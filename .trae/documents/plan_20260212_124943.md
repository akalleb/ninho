# Plano de Criptografia Estendida (Famílias e Crianças)

Este plano visa estender a criptografia de dados sensíveis para as tabelas de Famílias e Crianças, garantindo a proteção dos dados (LGPD) sem interromper o funcionamento atual do sistema (leitura/escrita).

## Objetivos
1.  **Criptografar dados sensíveis de Famílias:**
    *   Endereço completo, telefone, e-mail, renda mensal, observações.
2.  **Criptografar dados sensíveis de Crianças:**
    *   Diagnóstico, alergias, observações.
3.  **Implementar Hash Cego (Blind Index) para CPF/RG:**
    *   Permitir busca por CPF/RG sem revelar o dado real no banco.
    *   Isso será aplicado para Profissionais e Famílias.
4.  **Migração Segura:**
    *   Criar script para migrar dados existentes (criptografar o que está em texto plano).

## Passos de Implementação

### Fase 1: Atualização do Banco de Dados (Schema)
1.  **Criar colunas de hash:**
    *   Adicionar `cpf_hash` e `rg_hash` nas tabelas `professionals` e `families`.
    *   Isso permite buscar duplicidade sem descriptografar tudo.

### Fase 2: Atualização do Backend (Models e Lógica)
1.  **Implementar Hashing Seguro:**
    *   Adicionar função `get_data_hash` em `security.py` (HMAC-SHA256).
2.  **Atualizar Models (`models.py`):**
    *   Mapear novas colunas de hash.
3.  **Atualizar Endpoints (`main.py`):**
    *   **Famílias (Create/Update):** Criptografar campos sensíveis e gerar hashes.
    *   **Famílias (Read):** Descriptografar campos ao retornar para o frontend.
    *   **Crianças (Create/Update):** Criptografar diagnóstico, alergias, etc.
    *   **Crianças (Read):** Descriptografar na leitura.
    *   **Profissionais:** Atualizar para usar `cpf_hash` na verificação de duplicidade e criptografar o `cpf` real.

### Fase 3: Frontend (Transparente)
1.  **Ajuste de Leitura:**
    *   Garantir que o frontend receba os dados já descriptografados (via `computed_fields` do Pydantic no backend), para que nenhuma mudança complexa seja necessária nas telas React.

### Fase 4: Migração de Dados
1.  **Script de Migração:**
    *   Ler dados em texto plano.
    *   Gerar hash e criptografar.
    *   Salvar de volta no banco.

## Pontos Críticos (Para não quebrar o sistema)
*   **Fallback na Leitura:** Se o sistema tentar descriptografar algo que não está criptografado (dado antigo), ele deve retornar o dado original sem erro. (Já implementado na função `decrypt_data`).
*   **Busca:** A busca por CPF passará a usar a coluna `cpf_hash`.
